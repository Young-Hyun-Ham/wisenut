# 003. LangGraph Trace/Events 대시보드 재수정 피드백

## 1) 배경
- 기준 문서: `002_langgraph_trace_events_dashboard_spec.md`
- 1차 개발 결과 테스트에서 완료 판정 및 노드 가시성 관련 결함이 확인됨.
- 본 문서는 재작업 범위와 수용 기준을 명확히 정의한다.

## 2) 핵심 피드백 (반드시 수정)

### A. Completed 판정 오류 수정
현상:
- 실제 시나리오의 마지막 노드 종료 전, 중간 분기 노드 도달 시 `completed`로 상태가 변경됨.

요구사항:
- `completed` 상태는 "해당 run의 최종 종료 조건"을 만족할 때만 설정한다.
- 분기/라우팅/조건 노드(중간 제어 노드) 진입 또는 종료만으로 `completed`가 되면 안 된다.
- 최종 노드 종료 이벤트 또는 run 종료 이벤트를 기준으로 완료 판정 로직을 재구성한다.

구현 지침:
- 상태 전이(state transition)를 명시적으로 관리한다.
- 허용 전이 예시: `started -> running -> completed|failed|cancelled`
- `completed` 전이 조건을 단일 함수(예: `isRunCompleted(...)`)로 캡슐화해 분산 로직을 제거한다.
- 이벤트 역순 유입/지연 유입 시에도 완료 판정이 흔들리지 않도록 run별 마지막 유효 이벤트 시각 기준으로 보정한다.

### B. 노드 타입별 실행 이력 항목 추가
현상:
- 현재 대시보드에서 "어떤 노드 타입이 실행됐는지" 확인할 수 없음.

요구사항:
- run 상세 및 집계 영역에 `node_type` 기준 가시화를 추가한다.
- 최소 표시 항목:
  - `node_type`
  - `node_name`(또는 `node`)
  - `run_id`
  - `first_ts`, `last_ts`

구현 지침:
- 이벤트 원본 필드에 `node_type`이 없으면 매핑 규칙을 통해 유도값 생성(예: event_type/payload 기반).
- UI에서 node_type 필터를 추가한다.

### C. 노드별 실행 횟수 집계 추가
현상:
- 노드 단위 호출 빈도를 확인할 수 없음.

요구사항:
- 노드별 실행 횟수(count) 집계 기능을 추가한다.
- 최소 2개 뷰 제공:
  1. 현재 필터 범위 내 전체 집계(Top N)
  2. 선택한 run_id 내부 집계

권장 표시 컬럼:
- `node_name`
- `node_type`
- `execution_count`
- `error_count` (가능하면 포함)

## 3) 데이터 계약 보강
- 정규화 결과에 아래 필드를 반드시 포함:
  - `run_id`
  - `node`
  - `node_type`
  - `event_type`
  - `ts`
  - `status`
- `node_type` 미존재 시 `unknown`으로 명시하고, UI에서 숨기지 않는다.

## 4) 수용 기준 (DoD)
1. 분기 노드 이벤트만 발생한 상태에서는 `completed`로 전이되지 않는다.
2. 마지막 노드(또는 run 종료 이벤트) 처리 후에만 `completed`가 된다.
3. 대시보드에서 node_type별 실행 이력을 조회/필터할 수 있다.
4. 노드별 실행 횟수 집계가 전체/개별 run 기준으로 확인된다.
5. 회귀 테스트에서 기존 실패/취소 상태 판정이 깨지지 않는다.

## 5) 테스트 시나리오 (필수)
1. 정상 플로우 + 중간 분기 포함
- 기대값: 분기 노드 시점 `running` 유지, 최종 노드 종료 후 `completed`

2. 장시간 run + 이벤트 지연 유입
- 기대값: 지연 이벤트 수신 후에도 종료 판정이 잘못 뒤집히지 않음

3. node_type 누락 이벤트 포함
- 기대값: `unknown` 처리 + UI 노출

4. 동일 노드 반복 실행
- 기대값: node별 `execution_count` 정확히 누적

5. 실패 종료 run
- 기대값: `failed` 유지, completed 오판정 없음

## 6) 산출물
- 완료 판정 로직 수정 코드
- node_type 이력/필터 UI
- 노드별 실행 횟수 집계 UI/로직
- 테스트 증빙(로그 샘플 + 화면 캡처)
- 변경점 요약 문서(무엇이 왜 바뀌었는지)

## 7) 우선순위
1. P0: Completed 오판정 수정
2. P1: node_type 실행 이력 추가
3. P1: 노드별 실행 횟수 집계 추가
