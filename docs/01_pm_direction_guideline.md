# PM 방향성 작업방 운영 원칙

## 목적
이 작업방은 PM이 방향성을 제시하고, 개발자가 이를 기준으로 실제 소스를 수정할 수 있도록 문서를 제공하는 공간이다.

## 필수 원칙
1. 이 작업방에서는 소스 코드를 직접 수정하지 않는다.
2. 논의/제안/결정 사항은 반드시 `.md` 문서로 정리한다.
3. 문서는 개발자가 바로 작업 가능한 수준(요구사항, 변경 포인트, 기대 결과, 검증 방법 포함)으로 작성한다.
4. 문서는 `/docs` 폴더에 순번 기반 파일명으로 누적 작성한다.

## 문서 작성 형식 (권장)
- 배경/목표
- 범위(In/Out)
- 상세 요구사항
- 수정 대상 파일/모듈
- 구현 가이드
- 테스트/검증 기준
- 리스크 및 확인 필요 사항

## 파일명 규칙
- 예: `001_*.md`, `002_*.md`, `003_*.md`
- 동일 주제의 후속 문서는 다음 순번으로 추가한다.

## 현재 합의
- 본 작업방에서는 앞으로도 코드 수정 없이 문서만 작성한다.
